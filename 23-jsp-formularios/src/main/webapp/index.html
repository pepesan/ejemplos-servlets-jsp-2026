23 — JSP: Formularios con validación y PRG

Ciclo completo de un formulario web con validación servidor, re-renderizado de errores inline y patrón POST-Redirect-GET para evitar el doble envío.

Con errores:
GET /registro → formulario vacío
POST /registro → validación falla → forward al formulario con errores

Con éxito (PRG):
POST /registro → validación ok → sendRedirect → GET /registro?ok=1 → éxito

Formulario de registro

Campos: nombre (min 3), email (@), contraseña (min 6), edad (18-120).
Los errores aparecen inline junto al campo. Los valores válidos se conservan.
