13 — HttpServletRequest y HttpServletResponse

Los dos objetos que el contenedor pasa a cada método del servlet contienen toda la información de la petición y controlan completamente la respuesta. Este módulo los explora en detalle sin usar JSP.

InspeccionRequestServlet — ¿qué llega en cada petición?

GET /request

Muestra en tiempo real todo el contenido de HttpServletRequest:

línea de petición · parámetros de query string · cabeceras HTTP · datos del cliente · cookies · sesión

Sin parámetros Con query params

TiposRespuestaServlet — ¿qué puede devolver el servidor?

GET /respuesta?tipo=<valor>

El parámetro tipo selecciona qué tipo de respuesta construye el servlet:

200html 200json 200texto 200xml 201creado 302redirigir 404no-encontrado 500error 200cookie 200sesion

Flujo recomendado para explorar el módulo

  1. Visita GET /request y observa las cabeceras que envía tu navegador.
  2. Añade parámetros a la URL: ?nombre=Ana&ciudad=Madrid.
  3. Visita GET /respuesta?tipo=cookie para que el servidor establezca una cookie.
  4. Vuelve a GET /request y verifica que aparece la cookie en la sección 5.
  5. Visita GET /respuesta?tipo=sesion para guardar un atributo en sesión.
  6. Vuelve a GET /request y verifica que aparece el atributo en la sección 6.
  7. Prueba los distintos tipo con las DevTools del navegador (pestaña Network) para ver los status codes y cabeceras de respuesta.
