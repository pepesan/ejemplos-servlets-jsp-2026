47 — Struts 1.x: Subida de ficheros con FormFile

Struts integra CommonsMultipartRequestHandler (Apache Commons FileUpload) para procesar formularios multipart/form-data. El campo del fichero se declara como FormFile en el ActionForm; Struts lo rellena automáticamente igual que cualquier campo de texto.

Diferencia con el módulo 19

Módulo 19 (Servlet puro)Módulo 47 (Struts)
Config. del limite @MultipartConfig(maxFileSize=...) <controller maxFileSize="2M"/> en struts-config.xml
Acceso al fichero request.getPart("archivo") form.getArchivo() (campo FormFile)
Nombre original Parsear header Content-Disposition formFile.getFileName()
MIME type part.getContentType() formFile.getContentType()
Tamaño part.getSize() formFile.getFileSize()
Leer contenido part.getInputStream() formFile.getInputStream()

Piezas clave

// 1. ActionForm con campo FormFile
public class SubirFicheroForm extends ActionForm {
    private FormFile archivo;
    private String descripcion = "";
    // getters, setters, reset() — reset() debe asignar null al FormFile
}

// 2. struts-config.xml — límite de tamaño
<controller maxFileSize="2M"/>

// 3. JSP — enctype obligatorio, <html:file> en lugar de <input type="file">
<html:form action="/subir.do" method="post" enctype="multipart/form-data">
    <html:file property="archivo" accept="image/*,.pdf,.txt"/>
    <html:text property="descripcion"/>
    <html:submit value="Subir"/>
</html:form>

// 4. Action — leer FormFile
SubirFicheroForm f  = (SubirFicheroForm) form;
FormFile         ff = f.getArchivo();
String nombre = ff.getFileName();         // nombre original
String mime   = ff.getContentType();      // "image/jpeg", etc.
int    tam    = ff.getFileSize();         // bytes
