41 — Struts 1.x: CRUD en memoria

⚠️ Requiere JDK 8. Struts 1.3.x no es compatible con Java 9+.
CRUD completo de Contactos (crear, listar, editar, eliminar) usando cinco Actions distintos sobre un repositorio en memoria. Amplía el ejemplo 40 mostrando cómo un único ActionForm es compartido por varias Actions y cómo aplicar el patrón PRG con redirecciones.

Actions del módulo

URLMétodoActionResultado
/listar.do GET ListarAction Vista con la tabla de contactos
/ver.do?id=X GET VerAction Ficha de detalle del contacto
/nuevo.do GET NuevoAction Formulario vacío
/editar.do?id=XGETEditarAction Formulario con datos del contacto
/guardar.do POST GuardarAction Valida y crea/actualiza; redirige a la lista (PRG)
/eliminar.do POST EliminarActionBorra y redirige a la lista (PRG)

Flujo resumido

  GET /listar.do
     → ListarAction → request["contactos"] → lista.jsp

  GET /nuevo.do
     → NuevoAction → ContactoForm.reset() → formulario.jsp (vacío)

  GET /editar.do?id=3
     → EditarAction → carga Contacto(3) en ContactoForm → formulario.jsp (relleno)

  POST /guardar.do  nombre=Ana&email=ana@x.com&id=
     → GuardarAction → valida → guarda → redirect /listar.do   (PRG)

  POST /guardar.do  nombre=  (campo vacío)
     → GuardarAction → errores → forward formulario.jsp (valores preservados)

  POST /eliminar.do  id=3
     → EliminarAction → elimina → redirect /listar.do   (PRG)
