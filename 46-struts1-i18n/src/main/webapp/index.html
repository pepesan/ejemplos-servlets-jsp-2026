46 — Struts 1.x: Internacionalización (i18n)

Struts tiene soporte nativo de Locale: guarda el idioma elegido en sesión bajo la clave Globals.LOCALE_KEY y las taglibs (<bean:message>, <html:message>) seleccionan automáticamente el fichero de mensajes correspondiente (ApplicationResources_es.properties, ApplicationResources_en.properties, etc.).

Abrir demo (Español / English) →

Ficheros de mensajes

FicheroLocaleDescripción
ApplicationResources.properties por defecto Fallback cuando el locale no tiene fichero específico
ApplicationResources_es.properties es Español
ApplicationResources_en.properties en English

Cómo se cambia el idioma

CambiarIdiomaAction recibe ?lang=es o ?lang=en y guarda el Locale en sesión: 

// CambiarIdiomaAction.java
String lang = request.getParameter("lang");
request.getSession().setAttribute(Globals.LOCALE_KEY, new Locale(lang));
return mapping.findForward("inicio");  // redirect=true → PRG

Uso en JSP

<%@ taglib uri="http://struts.apache.org/tags-bean" prefix="bean" %>

<!-- Muestra el texto del locale activo -->
<bean:message key="app.titulo"/>

<!-- Con parámetros: saludo.resultado=¡Hola, {0}! -->
<bean:message key="saludo.resultado" arg0="Ana"/>

<!-- JSTL fmt también respeta el locale de Struts -->
<fmt:formatDate value="..." type="full" dateStyle="full"/>
