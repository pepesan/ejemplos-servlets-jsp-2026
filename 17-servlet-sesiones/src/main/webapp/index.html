17 — Sesiones HTTP

HttpSession permite asociar datos a un usuario concreto durante varias peticiones. El servidor asigna un identificador único (JSESSIONID) que el navegador devuelve mediante una cookie o parámetro de URL.

SesionServlet — ciclo de vida y API

Muestra el ID de sesión, fecha de creación, último acceso, timeout y atributos almacenados. Permite crear la sesión, añadir atributos e invalidarla. Tabla comparativa sesión vs cookie.

Abrir

ContadorServlet — visitas por usuario

Incrementa un contador guardado en sesión en cada petición. Dos navegadores distintos tienen contadores independientes. Demuestra que la sesión persiste entre peticiones.

Abrir

CarritoServlet — carrito de compra en sesión

Almacena un List<String> en sesión. Permite añadir artículos del catálogo, verlos persistir entre peticiones y vaciar el carrito con session.removeAttribute().
