26 — Recursos estáticos con JSP

Tomcat sirve automáticamente los ficheros de webapp/ mediante su servlet por defecto. Las JSP y los servlets pueden referenciarlos, pero la forma en que se construye la URL importa mucho.

Demos

Demo Qué muestra
demo-correcto.jsp CSS y JS cargados con ${pageContext.request.contextPath}. Funciona siempre.
demo-incorrecto.jsp Rutas relativas sin contextPath. Funciona accediendo directamente, falla si la URL cambia.
GET /api/demo (servlet → forward) ForwardServlet delega en una JSP dentro de WEB-INF/vistas/. La URL del navegador es /api/demo: las rutas relativas se rompen.
WEB-INF/privado.html Debe devolver 404: Tomcat bloquea el acceso directo a WEB-INF/.

Estructura de carpetas

webapp/
├── index.html            ← accesible en /
├── css/
│   └── estilos.css       ← accesible en /css/estilos.css
├── js/
│   └── demo.js           ← accesible en /js/demo.js
├── demo-correcto.jsp     ← accesible en /demo-correcto.jsp
├── demo-incorrecto.jsp   ← accesible en /demo-incorrecto.jsp
└── WEB-INF/
    ├── web.xml           ← nunca accesible desde el navegador
    ├── privado.html      ← nunca accesible desde el navegador
    └── vistas/
        └── forward.jsp   ← solo accesible via forward/include
