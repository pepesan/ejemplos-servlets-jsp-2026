45 — Gestión declarativa de excepciones

Struts 1.x permite declarar en struts-config.xml qué vista mostrar cuando una Action lanza una excepción, sin try/catch en el código Java. Es posible asignar handlers globales (para todas las Actions) o locales (para una Action concreta) y usar handlers personalizados que loggeen o enriquezcan el error antes de mostrar la vista.

Prueba los cuatro escenarios

Recurso no encontrado

Lanza RecursoNoEncontradoException (excepción de negocio propia). Interceptada por el handler global con LoggingExceptionHandler.

Probar →

Error de cálculo (local)

Lanza ArithmeticException (división por cero). Interceptada por el handler local de la action /producto, que sobrescribe el global.

Probar →

Error general

Lanza RuntimeException genérica. Interceptada por el handler global de RuntimeException con el ExceptionHandler por defecto de Struts.

Probar →

Sin excepción (camino feliz)

No se lanza ninguna excepción. La Action hace mapping.findForward("ok") normalmente.

Probar →

Configuración en struts-config.xml

<!-- GLOBALES: afectan a todas las Actions -->
<global-exceptions>
    <exception
        type="com.cursosdedesarrollo.RecursoNoEncontradoException"
        key="error.noEncontrado"
        path="/WEB-INF/vistas/noEncontrado.jsp"
        handler="com.cursosdedesarrollo.LoggingExceptionHandler"/>

    <exception
        type="java.lang.RuntimeException"
        key="error.general"
        path="/WEB-INF/vistas/error.jsp"/>
</global-exceptions>

<!-- LOCAL: solo para /producto, sobrescribe el global para ArithmeticException -->
<action path="/producto" type="com.cursosdedesarrollo.ProductoAction">
    <exception
        type="java.lang.ArithmeticException"
        key="error.calculo"
        path="/WEB-INF/vistas/errorCalculo.jsp"/>
    <forward name="ok" path="/WEB-INF/vistas/resultado.jsp"/>
</action>

Orden de resolución

Excepción lanzadaNivelHandlerVista
ArithmeticException Local (/producto) ExceptionHandler por defecto errorCalculo.jsp
RecursoNoEncontradoException Global LoggingExceptionHandler noEncontrado.jsp
RuntimeException Global ExceptionHandler por defecto error.jsp
¿Por qué ArithmeticException va al handler local y no al global RuntimeException?
Struts busca primero en la action, luego en global-exceptions. Dentro de cada nivel, busca la clase más específica (primero exacta, luego superclases). Al estar ArithmeticException declarada en la action, se encuentra antes de llegar al global.
