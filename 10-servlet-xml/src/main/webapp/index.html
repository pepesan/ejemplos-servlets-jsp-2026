10 — Servlet configurado por web.xml

Primer contacto con la tecnología Servlet: configuración clásica mediante el descriptor de despliegue WEB-INF/web.xml, sin ninguna anotación Java. Este era el único mecanismo disponible hasta Servlet 2.5 (Java EE 5).

Servlet disponible

HolaMundoXmlServlet

GET /hola-xml

Responde con texto plano. Mapeo declarado íntegramente en web.xml.

¿Cómo funciona el mapeo en web.xml?

El fichero WEB-INF/web.xml asocia una clase Java con una URL en dos pasos: primero se declara el servlet (nombre + clase), luego se mapea ese nombre a un patrón de URL. 

<servlet>
    <servlet-name>holaMundoXml</servlet-name>
    <servlet-class>com.cursosdedesarrollo.HolaMundoXmlServlet</servlet-class>
</servlet>

<servlet-mapping>
    <servlet-name>holaMundoXml</servlet-name>
    <url-pattern>/hola-xml</url-pattern>
</servlet-mapping>

web.xml vs @WebServlet

web.xml Servlet ≤ 2.5

  • Configuración centralizada
  • Funciona en cualquier versión
  • Necesario para override de anotaciones
  • Más verboso

@WebServlet Servlet 3.0+

  • Configuración junto al código
  • Menos ficheros
  • Requiere Tomcat 7+ / Java EE 6+
  • Ver módulo 11-servlet-anotaciones
