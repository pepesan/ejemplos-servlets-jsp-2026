42 — Struts 1.x: DispatchAction CRUD

⚠️ Requiere JDK 8. Struts 1.3.x no es compatible con Java 9+.
Mismo CRUD de Contactos que el módulo 41, pero con una única clase ContactoDispatchAction que agrupa todos los métodos CRUD. El parámetro method de la URL decide qué método se invoca.

Diferencia con el módulo 41

Módulo 41 — Actions separadosMódulo 42 — DispatchAction
6 clases Action distintas1 clase ContactoDispatchAction
6 <action> en struts-config.xml1 <action> con parameter="method"
/listar.do, /nuevo.do, …/contactos.do?method=listar, …
Lógica de validación en GuardarActionLógica de validación en ContactoDispatchAction.validar()

URLs disponibles

URLMétodo HTTPOperación
/contactos.do?method=listar GET Lista todos los contactos
/contactos.do?method=ver&id=XGET Detalle de un contacto
/contactos.do?method=nuevo GET Formulario vacío para crear
/contactos.do?method=editar&id=XGETFormulario relleno para editar
/contactos.do?method=guardar POST Crea o actualiza (PRG)
/contactos.do?method=eliminar&id=XPOSTElimina (PRG)

Cómo despacha DispatchAction

  GET /contactos.do?method=editar&id=3
        │
        ▼ ActionServlet → busca 
        │   crea ContactoForm, llama reset(), setId("3")
        │   lee request.getParameter("method") → "editar"
        │   invoca ContactoDispatchAction.editar(mapping, form, req, res)
        ▼
  ContactoDispatchAction.editar()
        │   carga Contacto(3) del repositorio
        │   sobreescribe campos del form
        │   findForward("formulario")
        ▼
  formulario.jsp
