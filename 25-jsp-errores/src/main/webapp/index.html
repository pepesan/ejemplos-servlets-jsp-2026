25 — Manejo de errores en JSP

Dos mecanismos complementarios para gestionar errores en una aplicación web Java: la directiva errorPage (por página JSP) y las entradas <error-page> de web.xml (global).

Mecanismo 1 — Directiva errorPage
Cada JSP puede declarar %@ page errorPage="/mi-error.jsp" %. Si lanza una excepción no capturada, el contenedor hace forward a esa JSP, que debe tener isErrorPage="true" para acceder al objeto implícito exception.
Mecanismo 2 — <error-page> en web.xml
Configuración global para códigos HTTP (404, 500…) y tipos de excepción. Se activa cuando el contenedor recibe un sendError() o una excepción no capturada de un servlet o JSP sin errorPage.

Mecanismo 2 — Páginas de error globales (web.xml)

Errores HTTP — sendError()

El servlet llama a resp.sendError(código, mensaje). El contenedor busca la entrada <error-code> en web.xml.

404 Not Found 500 Internal Error 404 real (URL inexistente)

Excepciones no capturadas en servlet

El servlet lanza una excepción. El contenedor usa la entrada <exception-type> más específica de web.xml.

RuntimeException NullPointerException ArithmeticException
