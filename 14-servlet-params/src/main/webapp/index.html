14 — Parámetros GET y POST

Los servlets leen parámetros de dos fuentes distintas: la query string de la URL (método GET) y el cuerpo de la petición de un formulario (método POST). La API es idéntica en ambos casos: req.getParameter().

GETParámetros en la URL: /get?nombre=Ana&edad=30. Visibles, marcables, cacheables. Límite ~2 KB.
POSTParámetros en el cuerpo HTTP. Invisibles en la URL. Sin límite práctico. Necesario para datos sensibles o ficheros.

ParamsGetServlet — query string

GET /get

Lee todos los parámetros de la query string. Demuestra valores múltiples para el mismo nombre.

Sin parámetros nombre + ciudad multivalor (color×3) URL encoding

ParamsPostServlet — formulario HTML

GET /post  →  POST /post

GET devuelve el formulario. POST recibe y muestra los campos enviados. Abre las DevTools (F12 → Network → Payload) para ver el cuerpo de la petición.

Abrir formulario

Flujo de aprendizaje

  1. Visita GET /get sin parámetros y observa el resultado.
  2. Añade ?nombre=Ana&ciudad=Madrid a la URL.
  3. Prueba el parámetro multivalor: color×3.
  4. Abre GET /post y rellena el formulario. Observa que la URL no cambia.
  5. Abre las DevTools, ve a Network y vuelve a enviar el formulario para ver el cuerpo POST.
  6. Comprueba que getParameter() funciona igual para GET y POST.
