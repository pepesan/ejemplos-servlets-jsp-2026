11 — Servlet configurado por @WebServlet Servlet 3.0+

A partir de Servlet 3.0 (Java EE 6 / Tomcat 7) es posible registrar un servlet directamente con la anotación @WebServlet, eliminando la necesidad de declararlo en web.xml.

Servlet disponible

HolaMundoServlet

GET /hola

Responde con texto plano. Mapeo declarado con @WebServlet("/hola").

¿Cómo funciona la anotación?

La anotación @WebServlet reemplaza el bloque <servlet> + <servlet-mapping> del web.xml. El contenedor la detecta automáticamente en el arranque. 

@WebServlet("/hola")                  // una URL
@WebServlet({"/hola", "/hello"})      // varias URLs
@WebServlet(
    urlPatterns = "/hola",
    name        = "holaMundo",
    initParams  = {
        @WebInitParam(name = "autor", value = "Curso 2026")
    }
)
public class HolaMundoServlet extends HttpServlet { ... }

Atributos principales de @WebServlet

AtributoEquivalente en web.xmlDescripción
urlPatterns / value<url-pattern>Una o varias URLs que atiende el servlet
name<servlet-name>Nombre interno del servlet (opcional)
initParams<init-param>Parámetros de inicialización
loadOnStartup<load-on-startup>Carga eagerly al arrancar el servidor

Cuándo seguir usando web.xml

Las anotaciones no pueden sobreescribirse sin recompilar. En proyectos con múltiples entornos (dev / prod) o cuando el mapeo de URLs lo decide el equipo de operaciones, web.xml sigue siendo necesario. Ver módulo 10-servlet-xml.
