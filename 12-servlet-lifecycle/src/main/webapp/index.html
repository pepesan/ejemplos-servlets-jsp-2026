12 — Ciclo de vida del Servlet

El contenedor (Tomcat) gestiona el ciclo de vida completo de cada servlet: crea la instancia, la inicializa, la usa para atender peticiones y la destruye. El programador solo implementa los tres métodos del ciclo de vida.

Diagrama del ciclo de vida

Arranque del servidor
    │
    ▼
┌─────────────┐
│   init()   │  ← UNA VEZ  Lee init-params, abre conexiones
└──────┬──────┘
         │
         ▼
┌─────────────┐
│ service()  │  ← CADA PETICIÓN  Puede haber hilos concurrentes
│ doGet/Post │
└──────┬──────┘
         │
         ▼
┌─────────────┐
│ destroy()  │  ← UNA VEZ  Libera recursos al parar el servidor
└─────────────┘

Servlets de este módulo

CicloVidaServlet — ciclo de vida completo

GET /ciclo-vida

Sobreescribe init(), service() y destroy(). Muestra en tiempo real cuándo se ejecutó init(), cuántas peticiones se han atendido (contador compartido con AtomicInteger) y los init-param leídos del web.xml. Recarga la página varias veces para ver el contador subir; para el servidor para ver destroy() en la consola.

FormularioServlet — doGet + doPost

GET /formulario — muestra el formulario
POST /formulario — procesa los datos (enviar desde el formulario)

Un mismo servlet atiende dos métodos HTTP: doGet() sirve el formulario HTML; doPost() lee los parámetros del cuerpo de la petición, valida y devuelve un resumen. Aplica el patrón PRG (Post-Redirect-Get) para evitar el reenvío al recargar con F5. También demuestra el escape de HTML para prevenir XSS.

Conceptos clave

ConceptoDónde verlo
init(ServletConfig) — una vez Consola de Tomcat al arrancar + página /ciclo-vida
service() → doGet() / doPost() — cada petición Contador en /ciclo-vida, formulario en /formulario
destroy() — una vez Consola de Tomcat al parar el servidor (Ctrl+C)
load-on-startup web.xml de este módulo: CicloVidaServlet carga al arrancar
init-param Parámetros autor y version visibles en /ciclo-vida
Patrón PRG Enviar el formulario vacío → redirección 302 a /formulario?error=vacio
Escape XSS Escribe <script> en el nombre del formulario → se muestra como texto
